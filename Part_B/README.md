# Часть B: Nginx контейнер на пользовательском порту

**Основные требования:**
- Имя контейнера: `servidor_web`
- Порт хоста: 8181
- Порт контейнера: 80
- Образ: nginx

---

## Задача 1: Создание контейнера и проверка статуса

**Статус:** ❌ Не выполнена

### Требуемые скриншоты:
- [ ] Команда создания контейнера с выводом
- [ ] Вывод `docker ps` подтверждающий работу контейнера

### Команды для выполнения:
```bash
# Создание и запуск nginx контейнера
docker run -d \
  --name servidor_web \
  -p 8181:80 \
  nginx

# Проверка что контейнер запущен
docker ps

# Дополнительная информация о контейнере
docker inspect servidor_web
```

### Ожидаемый результат:
- Контейнер `servidor_web` запущен
- Порт 8181 пробрасывается на порт 80 контейнера

---

## Задача 2: Доступ через веб-браузер

**Статус:** ❌ Не выполнена

### Требуемые скриншоты:
- [ ] Окно браузера с адресом http://localhost:8181 или http://<IP>:8181
- [ ] Видна страница приветствия nginx (Welcome to nginx!)

### Как проверить:
1. Откройте веб-браузер
2. Перейдите на `http://localhost:8181`
3. Если браузер на другой машине, используйте IP адрес Docker хоста
4. После успешного подключения сделайте скриншот

### Получение IP адреса:
```bash
# IP адрес Вашего Docker хоста
hostname -I
```

## Задача 4: Удаление контейнера

**Статус:** ❌ Не выполнена

### Требуемые скриншоты:
- [ ] Команда `docker stop servidor_web`
- [ ] Вывод `docker ps -a` с остановленным контейнером
- [ ] Команда `docker rm servidor_web`
- [ ] Вывод `docker ps -a` подтверждающий удаление контейнера

### Команды для выполнения:
```bash
# 1. Остановка контейнера
docker stop servidor_web

# 2. Проверка что он остановлен
docker ps -a | grep servidor_web

# 3. Удаление контейнера
docker rm servidor_web

# 4. Проверка что он удален
docker ps -a | grep servidor_web  # Не должно быть результата
```

### Примечания:
- ВАЖНО: Контейнер ДОЛЖЕН быть остановлен перед удалением
- Если нет - используйте `docker rm -f servidor_web` (принудительное удаление)

---

## Дополнительные команды для работы

```bash
# Просмотр логов контейнера
docker logs servidor_web

# Остановка без удаления
docker stop servidor_web

# Перезапуск контейнера
docker start servidor_web

# Вход в работающий контейнер
docker exec -it servidor_web bash

# Информация о контейнере
docker inspect servidor_web

# Статистика контейнера
docker stats servidor_web
```
